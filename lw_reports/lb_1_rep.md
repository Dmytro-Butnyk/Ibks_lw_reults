# Лабораторна робота № 1

**Тема:** Арифметичні операції над багато розрядними цілими числами

**Мета:** набуття практичних навичок реалізації на базовому наборі команд процесора операцій «довгої» арифметики над цілими числами

## Порядок виконання

1. **Аналіз завдання та підготовка середовища:** Для виконання роботи було обрано архітектуру x86-64 та середовище Rider з підтримкою макроасемблера MASM. Основним завданням є реалізація арифметичних операцій для 256-бітних чисел, які представлені у пам'яті як масиви з чотирьох 64-бітних слів (QWORD).

2. **Реалізація базових арифметичних функцій (Assembly):**
   Було розроблено модуль на мові асемблера, що використовує команди `ADD/ADC` для додавання та `SUB/SBB` для віднімання з урахуванням переносу/займу між 64-бітними регістрами. Також реалізовано функції порівняння, інкременту та побітових зсувів для 256-бітних структур.

3. **Розробка керуючого додатку (C++):**
   Написано програму на C++, яка викликає асемблерні процедури та забезпечує введення/виведення даних. Для перевірки правильності обчислень використано тестові набори даних ($0, 1, 2, 2^n$).

4. **Тестування та верифікація:**
   Програма була скомпільована та запущена. Результати обчислень у шістнадцятковому форматі було порівняно з теоретичними очікуваннями.

## Висновок
У ході лабораторної роботи було опановано методи реалізації «довгої» арифметики для цілих чисел великої розрядності (256 біт). Практично застосовано команди процесора Intel64 для роботи з переносами та маніпуляції даними у пам'яті. Розроблений програмний комплекс демонструє можливість виконання точних розрахунків для чисел, що виходять за межі стандартних типів даних.

## Відповіді на контрольні запитання

1. **Розширений алгоритм Евкліда:**
   Це алгоритм, який знаходить не лише найбільший спільний дільник (НСД) двох цілих чисел $a$ та $b$, але й цілі числа $x$ та $y$ (коефіцієнти Безу), такі що $ax + by = НСД(a, b)$. Він критично важливий для знаходження мультиплікативного оберненого елемента за модулем у криптографії.

2. **Методи і метрики оцінки обчислювальної складності:**
   - **Методи:** Асимптотичний аналіз (використання нотації $O$), емпіричне вимірювання часу виконання.
   - **Метрики:** Часова складність (кількість операцій залежно від вхідних даних $n$) та просторова складність (обсяг необхідної пам'яті).
