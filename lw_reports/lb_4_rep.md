# Лабораторна робота № 4

**Тема: Функції хешування**

**Мета:** освоїти на практиці використання функцій хешування SHA-2.

## Порядок виконання

1. **Розробка консольного додатка:** Написано програму мовою C++ у середовищі Rider для обчислення хешу файлу за алгоритмом SHA-256. Реалізовано отримання імені вхідного файлу з параметрів командного рядка.
2. **Відображення на пам'ять (Memory-Mapped Files):** Згідно з рекомендаціями, замість стандартного побайтового читання використано механізм відображення файлу в пам'ять через Windows API (`CreateFileMapping`, `MapViewOfFile`). Це дозволяє працювати з вмістом файлу безпосередньо як з масивом у пам'яті, що суттєво підвищує швидкість обробки великих файлів.
3. **Реалізація алгоритму SHA-256 (SHA-NI):** Для обробки 512-бітних (64-байтових) блоків повідомлення застосовано апаратні інструкції процесора Intel SHA Extensions (SHA-NI) за допомогою інтринсиків `<immintrin.h>`.
4. **Доповнення (Padding):** Реалізовано алгоритм доповнення повідомлення. До кінця файлу додається біт `1` (байт `0x80`), необхідна кількість нулів та 64-бітне число, що позначає початкову довжину повідомлення у бітах (у форматі big-endian).
5. **Збереження результату:** Отриманий 256-бітний хеш конвертується у шістнадцятковий рядок (64 символи) та зберігається у новий файл, ім'я якого формується додаванням розширення `.sha256` до імені оригінального файлу.

## Висновок
Під час виконання лабораторної роботи було на практиці освоєно використання функцій хешування сімейства SHA-2. Розроблено високопродуктивний додаток мовою C++, який реалізує алгоритм SHA-256 за допомогою апаратного розширення SHA-NI та механізму відображення файлів на пам'ять (Memory-Mapped Files). Засвоєно принципи доповнення повідомлень (padding) та формування фінального дайджесту.

## Відповіді на контрольні запитання

1. **Які шість варіантів алгоритмів входять до сімейства SHA-2?**
   До сімейства входять: SHA-224, SHA-256, SHA-384, SHA-512, SHA-512/224, SHA-512/256.
2. **Який розмір блоку повідомлення для SHA-256 та SHA-512?**
   Для SHA-256 розмір блоку повідомлення становить 512 біт (64 байти). Для SHA-512 — 1024 біти (128 байт).
3. **Як виконується доповнення (Padding) повідомлення?**
   Повідомлення доповнюється так, щоб його довжина стала кратною розміру блоку (512 біт для SHA-256). Спочатку в кінець повідомлення додається одиничний біт (зазвичай це байт `0x80`), потім серія нульових бітів, і в самому кінці — 64-бітне число у форматі big-endian (Network Byte Order), яке містить довжину вихідного повідомлення в бітах.
4. **Які три основні інструкції SHA-NI використовуються для SHA-256?**
   Основними інструкціями є: `SHA256RNDS2` (виконує 2 раунди хешування), `SHA256MSG1` та `SHA256MSG2` (виконують обчислення наступних подвійних слів розкладу повідомлення).
5. **Які регістри використовує SHA-NI?**
   Інструкції SHA-NI використовують 128-бітні XMM-регістри (SIMD).
6. **Чи підтримує SHA-NI алгоритм SHA-512?**
   Ні, набір команд SHA-NI не підтримує алгоритм SHA-512. Для апаратного прискорення SHA-512 зазвичай використовують розширення AVX-512 (зокрема інструкції AVX-512 IFMA).
7. **Як перевірити підтримку SHA-NI у програмі?**
   Підтримка перевіряється за допомогою виконання інструкції `cpuid` при `EAX = 7` і `ECX = 0`. Якщо після виконання 29-й біт у регістрі `EBX` встановлений (дорівнює 1), то розширення SHA-NI доступне.
8. **Що таке PQC (Post-Quantum Cryptography)?**
   Постквантова криптографія (PQC) — це криптографічні алгоритми, які розробляються для забезпечення стійкості до криптоаналізу з боку майбутніх квантових комп'ютерів. Квантові комп'ютери здатні ефективно зламувати сучасні асиметричні алгоритми (як RSA чи алгоритми на еліптичних кривих), проте симетричні шифри та хеш-функції (включно із SHA-256) вважаються відносно стійкими до квантових атак, вимагаючи лише збільшення довжини ключа/хешу для збереження безпеки.
