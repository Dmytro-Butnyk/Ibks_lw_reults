# Лабораторна робота № 3

**Тема: Симетрична криптосистема AES**

**Мета:** на практиці освоїти шифрування та дешифрування повідомлень за допомогою криптосистеми AES.

## Порядок виконання

1. **Аналіз алгоритму та інструментів:** Ознайомлено з теоретичними відомостями щодо алгоритму AES. Оскільки розмір блоку фіксовано на рівні 128 біт (16 байт), для апаратного прискорення було використано набір інструкцій AES-NI процесорів архітектури x86-64, які працюють з 128-бітними регістрами XMM.
2. **Розробка консольного додатка:** Написано програму мовою C++ у середовищі Rider. Реалізовано обробку аргументів командного рядка у форматі `aescipher.exe <-enc | -dec> <input file> <output file> <key>`.
3. **Генерація раундових ключів (Key Expansion):** Для розширення початкового 128-бітного ключа до 11 раундових ключів використано інтринсики `_mm_aeskeygenassist_si128`. Для дешифрування застосовано `_mm_aesimc_si128` для конвертації раундових ключів (зворотне перетворення стовпців).
4. **Шифрування (Режим ECB):** Дані з файлу зчитуються блоками по 16 байт. Згідно з інструкцією, якщо розмір файлу не кратний 16 байтам, останній блок доповнюється нулями (Zero Padding). Виконуються 10 раундів: перший `_mm_xor_si128`, 9 раундів `_mm_aesenc_si128` та фінальний `_mm_aesenclast_si128`.
5. **Дешифрування:** Зворотній процес, який використовує зчитані блоки по 16 байт, 9 раундів `_mm_aesdec_si128` і фінальний раунд `_mm_aesdeclast_si128`.
6. **Робота з файлами:** Читання та запис оптимізовано шляхом повного завантаження файлу в пам'ять (імітація Memory Mapping).

## Висновок
Під час виконання лабораторної роботи було розроблено консольний додаток мовою C++ для симетричного шифрування та дешифрування файлів алгоритмом AES-128 у режимі ECB. Для досягнення максимальної продуктивності успішно застосовано апаратні інструкції розширення AES-NI (intrinsics). Опрацьовано принципи блокового шифрування та заповнення неповних блоків нулями (padding).

## Відповіді на контрольні запитання

1. **Порівняльна характеристика симетричних і асиметричних криптосистем.**
   Симетричні криптосистеми (наприклад, AES) використовують один і той самий ключ для шифрування та дешифрування; вони швидші і підходять для великих даних. Асиметричні (RSA) використовують пару ключів; вони повільніші і зазвичай застосовуються для обміну ключами та цифрових підписів.

2. **Яка фіксована довжина блоку в алгоритмі AES і які варіанти довжини ключа підтримує стандарт?**
   Розмір блоку фіксований — 128 біт (16 байт). Стандарт підтримує ключі довжиною 128, 192 та 256 бітів.

3. **Скільки ітерацій (раундів) перетворень виконується для різних ключів?**
   Для ключа 128 біт — 10 раундів, для 192 біт — 12 раундів, для 256 біт — 14 раундів.

4. **Який з раундів AES забезпечує нелінійність шифру?**
   Нелінійність забезпечується етапом **SubBytes** (заміна байтів через S-box).

5. **Чому AES-NI оперує саме 128-бітними регістрами?**
   Тому що розмір блоку AES дорівнює 128 бітам. Використання AVX-512 дозволяє шифрувати кілька блоків паралельно, що дає приріст швидкості в режимах типу ECB або CTR.

6. **Чим останній раунд AES відрізняється від попередніх?**
   В останньому раунді відсутня операція **MixColumns**. Це зроблено для спрощення апаратної реалізації та симетрії алгоритмів шифрування/дешифрування.
