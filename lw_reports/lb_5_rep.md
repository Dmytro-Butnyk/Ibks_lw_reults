# Лабораторна робота № 5

**Тема: Електронний цифровий підпис**

**Мета:** освоїти на практиці використання електронного цифрового підпису.

## Порядок виконання

1. **Аналіз завдання та вибір алгоритмів:** Для реалізації ЕЦП було обрано комбінацію алгоритму хешування **SHA-256** (для забезпечення цілісності) та асиметричного алгоритму **RSA-2048** (для автентичності). Програма розроблена як консольний додаток `hash.exe`.
2. **Використання апаратних засобів та API:** Обчислення хешу та операції з ключами RSA реалізовані за допомогою **Windows Crypto API NG (CNG)** з обов'язковим використанням PKCS#1 padding.
3. **Робота з файлами (Memory Mapping):** Читання файлів реалізовано через проектування файлів у пам'ять (`CreateFileMapping`, `MapViewOfFile`), що прискорює обробку великих обсягів даних.
4. **Реалізація підпису (Режим hash) та експорт ключа:**
   * Програма генерує пару ключів RSA-2048.
   * Обчислюється SHA-256 хеш вхідного файлу та підписується закритим ключем RSA.
   * Отриманий підпис (256 байт) записується в кінець вихідного файлу.
   * Згенерована пара ключів (Private/Public Key BLOB) експортується та зберігається у вказаний користувачем файл (наприклад, `private.key`).
5. **Реалізація перевірки (Режим test) та імпорт ключа:**
   * Програма зчитує файл ключа та імпортує його у провайдер CNG.
   * Зчитуються останні 256 байт цільового файлу (підпис).
   * Обчислюється хеш решти файлу (даних).
   * За допомогою імпортованого ключа RSA перевіряється, чи відповідає підпис обчисленому хешу.
6. **Тестування:** Тестування підтвердило, що підпис успішно валідується при використанні збереженого ключа. Зміна файлу ключа або хоча б одного байта підписаного файлу призводить до статусу "INVALID".

## Отримання КЕП (Інструкція на прикладі Приват24)

Згідно з пунктом 3 завдання, було отримано кваліфікований електронний підпис. Процес складається з таких етапів:

1. Авторизація в системі **Дія**.
2. Перехід у меню **"Дія.Підпис"** -> **"Активувати Дія.Підпис"**.
3. Перевірка персональних даних (ПІБ, адреса, паспортні дані).
4. Встановлення пароля на файл ключа (КЕП).

## Висновок
Під час виконання роботи було практично засвоєно механізм створення та перевірки електронного цифрового підпису. Програму вдосконалено можливістю експорту та імпорту ключів RSA у бінарні файли, що дозволяє проводити незалежну автентифікацію. Підтверджено, що ЕЦП забезпечує одночасно цілісність документа та ідентифікацію автора, а зміна даних миттєво робить підпис недійсним.

## Відповіді на контрольні запитання

1. **Задачі і алгоритми ЕЦП.**
   Основними задачами ЕЦП є: забезпечення цілісності (захист від змін), автентифікація автора та невідмовність (автор не може заперечити факт підпису). Основні алгоритми: RSA, DSA, ECDSA (на еліптичних кривих), Ель-Гамаля (EGSA).
2. **На чому ґрунтується криптостійкість алгоритмів RSA і EGSA?**
   Криптостійкість RSA базується на складності задачі факторизації (розкладання на множники) великих цілих чисел. Стійкість EGSA (Ель-Гамаля) базується на складності задачі обчислення дискретного логарифму у скінченному полі.
3. **Порівняльна характеристика алгоритмів RSA і EGSA.**
   RSA швидший при перевірці підпису, але повільніший при генерації ключів. EGSA (і його похідна DSA) забезпечує менший розмір підпису при аналогічній стійкості, але потребує використання якісного генератора випадкових чисел для кожного підпису.
4. **Алгоритм DSA.**
   DSA (Digital Signature Algorithm) — це американський стандарт цифрового підпису, розроблений NIST на основі алгоритму Ель-Гамаля. Він використовується виключно для створення підпису (не для шифрування даних) і базується на складності дискретного логарифмування.
